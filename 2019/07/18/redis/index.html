redis

简介

REmote DIctionary Server(Redis) 是一个由Salvatore Sanfilippo写的key-value存储系统。

Redis是一个开源的使用ANSI C语言编写、遵守BSD协议、支持网络、可基于内存亦可持久化的日志型、Key-Value数据库，并提供多种语言的API。

它通常被称为数据结构服务器，因为值（value）可以是 字符串(String), 哈希(Map), 列表(list), 集合(sets) 和 有序集合(sorted sets)等类型。

Redis 是完全开源免费的，遵守BSD协议，是一个高性能的key-value数据库。

Redis 与其他 key - value 缓存产品有以下三个特点：

  • Redis支持数据的持久化，可以将内存中的数据保持在磁盘中，重启的时候可以再次加载进行使用。
  • Redis不仅仅支持简单的key-value类型的数据，同时还提供list，set，zset，hash等数据结构的存储。
  • Redis支持数据的备份，即master-slave模式的数据备份。

与其他key-value区别

  • Redis有着更为复杂的数据结构并且提供对他们的原子性操作，这是一个不同于其他数据库的进化路径。Redis的数据类型都是基于基本数据结构的同时对程序员透明，无需进行额外的抽象。
  • Redis运行在内存中但是可以持久化到磁盘，所以在对不同数据集进行高速读写时需要权衡内存，应为数据量不能大于硬件内存。在内存数据库方面的另一个优点是， 相比在磁盘上相同的复杂的数据结构，在内存中操作起来非常简单，这样Redis可以做很多内部复杂性很强的事情。 同时，在磁盘格式方面他们是紧凑的以追加的方式产生的，因为他们并不需要进行随机访问。

优势

  • 性能极高 – Redis能读的速度是110000次/s,写的速度是81000次/s 。
  • 丰富的数据类型 – Redis支持二进制案例的 Strings, Lists, Hashes, Sets 及 Ordered Sets 数据类型操作。
  • 原子 – Redis的所有操作都是原子性的，同时Redis还支持对几个操作全并后的原子性执行。
  • 丰富的特性 – Redis还支持 publish/subscribe, 通知, key 过期等等特性。

安装

windows

zip解压即用，需要gcc环境

redis.conf为配置文件，可按需更改

运行： redis-server.exe redis.conf

客户端: redis-cli.exe -h 127.0.0.1 -p 6379

linux

make完成后，即会在用户目录下出现服务程序和客户端程序

启动: $ ./redis-server

带配置启动: $./redis-server redis.conf

客户端: ./redis-cli -h 127.0.0.1 -p 6379

配置文件说明

  1. daemonize no

    是否以守护进程方式运行，默认否，可修改为yes启用守护线程

  2. pidfile /var/run/redis.pid

    当开启了守护线程后，默认hi把pid写入/var/run/redis.pid 依然可以通过修改本项指定

  3. port 6379

    指定监听端口，默认即6379

  4. bind 127.0.0.1

    绑定的主机地址，127.0.0.1即本机，同localhost

  5. timeout 300

    客户端闲置时间，如果指定为0即不关闭连接

  6. loglevel verbose

    日志记录级别，debug\verbose\notice\warning

  7. logfile stdout

    日志记录方式，默认标准输出，若redis以守护进程进行，而本项又为标准输出，则日志将会出现在/dev/null

  8. databases 16

    指定数据库数量，默认数据库为0，使用SELECT DBID 即可在连接上指定数据库id

  9. save

    时间 更新操作数决定持久化时机

    save 900 1： 900秒1个更改即持久化

    save 300 10： 300秒10个更改即持久化

    save 60 10000： 60秒10000个更改即持久化

  10. rdbcompression yes

    指定存储至本地数据时是否压缩数据，默认yes，redis采用的压缩技术为LZF，若需要节省CPU处理时间，可以修改为no，会导致数据库文件变大，依照需要考虑时间和空间的换算

  11. dbfilename dump.rdb

    指定本地数据库文件名，默认即dump.rdb

  12. dir ./

    指定本地数据库存放目录

  13. slaveof

    设置本机为slav服务时选哟设置master服务的ip及端口，redis启动后会从master获取数据进行同步

  14. masterauth

    当master服务设置了密码保护，slav服务也需要提供密码

  15. requirepass foobared

    设置redis连接密码，如果配置了此项，客户端在连接redis后进行操作需要通过auth 命令提供密码进行认证，默认关闭

  16. maxclients 128

    设置同一时间最大客户端连接数，默认无限制，即设置为0，当客户端连接数达到限制时，redis会关闭新连接并将max number of clinets reached返回给客户端

  17. maxmemory

    指定redis最大内存限制，redis会在启动时将数据加载进内存，当到达内存峰值时，redis会尝试清除到期或即将到期的key，若此方法未能解决，将无法再进行写入操作，但依旧可以读取，redis的新vm机制会将key存放再内存中，value放进swap区

  18. appendonly no

    指定是否再每次更新操作后进行日志记录，redis默认是异步将数据写入磁盘，若不开启，断电时会导致数据丢失，redis本身同步数据四按照save条件同步，若数据还未被持久化时发生断电即会丢失，默认不开启

  19. appendfilename appendonly.aof

    指定更新的日志文件名，默认即appendonly.aof

  20. appendfsync everysec

    指定更新日志条件

    no:由操作系统进行数据缓存持久化

    always:每次更新后都手动调用fsync进行持久化

    everysec:每秒持久化一次

  21. vm-enabled no

    是否启用虚拟内存机制，默认不启用，vm机制将数据分页存放，redis将访问量较少的数据swap到磁盘上，访问多的数据由磁盘放在内存中

  22. vm-swap-file /tmp/redis.swap

    虚拟内存文件路径，默认/tmp/redis.swap

    多个redis不共享

  23. vm-max-memory 0

    将所有大于vm-max-memory的数据存入虚拟内存，无论vm-max-memory设置的多小，所有的所有数据都是由内存存储的，当vm-max-memory设置为0时间四行就是所有value都存于磁盘中,默认即为0

  24. vm-page-size 32

    redis swap文件分了很多page，一个对象可以保存在多个page上面，但一个page不能被多个对象共享，vm-page-size根据存储的数据大小设定，如果存储很多小对象，page设置为32或64，如果对象比较大，则增加page，如果不确定，使用默认值即可

  25. vm-pages 134217728

    设置swap文件中的page数量，由于页表是存放于内存中的，在磁盘上每8个pages将小号1byte的内存

  26. vm-max-threads 4

    设置访问swap文件的线程数，不超过处理器的核心数即可，若为0，那么所有对swap的操作都是串行的，会造成阻塞，默认为4

  27. glueoutputbuf yes

    向客户端应答时，是否将较小的包合并为一个包发送，默认开启

  28. hash-max-zipmap-entries 64

  29. hash-max-zipmap-value 512

    超过一定数量或最大元素超过临界值时的哈希算法

  30. activerehashing yes

    是否激活重置哈希，默认开启

  31. include /path/to/local.conf

    是否包含其他配置文件，用于在同一主机上多个redis实例之间使用共同的配置文件，同时哥哥实例又需要自己特有的配置文件

数据类型

String

​ String是redis最基本的类型，一个key一个value。

​ String是二进制安全的，可以包含任何数据，例如图片或者序列化后的对象。

​ String的一个键最大能保存512MB大小

Hash

​ Redis Hash是键值对的集合。

​ 类似于String类型的field和value的映射表，hash适合用于存储对象。

List

​ Redis List是简单的字符串列表，按照插入的顺序进行排序，可以添加元素在头（左）或尾（右）

​ 列表最多可以存储2^(32-1)个元素，40多亿。

Set

​ Redis Set是String的无序列表，由于是通过哈希表实现的 ，所以添加，删除，查询的复杂度都是O(1)

Zset

​ Redis Zset和set一样也是String的列表，不允许重复。

​ 不同的是每个元素都会关联一个double的分数，用于排序，zset的成员唯一，但分数可以重复。

发布订阅

发布订阅(pub/sub)时一种消息通信模式，发送者(pub)发送消息，订阅者(sub)接受消息。

redis客户端可以订阅任意数量的频道，当有新消息通过PUBLISH命令发送给频道时，这个消息会被发送到订阅者的客户端上。

事务

事务可以一次执行多个命令并有如下特性：

  • 事务中所有命令都会序列化，并且按顺序执行，执行过程中不会被其他客户端的命令请求打断。
  • 事务是原子操作，要么全部执行，要么全部失败

过程：

  • 开始事务
  • 命令入队
  • 执行事务
命令 描述
DISCARD 取消事务，放弃执行事务
EXEC 执行所有事务
MULTI 标记事务块的开始
UNWATCH 取消对所有key的监视
WATCH key1 [key2…] 监视一个或多个key，如果事务执行前这些key被修改，那么事务会失败

性能测试

​ 命令 redis-benchmark [option] [option value]

参数 描述 默认
-h 指定服务器主机 127.0.0.1
-p 指定端口 6379
-s 指定服务器socket
-c 指定并发连接数 50
-n 指定请求数 10000
-d 以字节的形式指定SET/GET值的数据大小 2
-k 1=keep alive 0=reconnect 1
-r SET/GET/INCR 使用随机key，SADD使用随机值
-p 通过管道传输请求 1
-q 强制退出redis，仅显示quert/sec值
–csv 以CSV格式退出
-l 生成循环，永久性执行测试
-t 仅运行以逗号分隔的测试命令列表

客户端

​ redis通过监听一个tcp端口或Unix socket的方式接收客户端的连接。

​ 首先客户端socket会被设置为非阻塞模式，redis在网络事件的处理上使用的模型为非阻塞多路复用模型。为socket设置TCP_NODELAY属性，禁用Nagle算法，然后创建可读文件事件用于监听客户端socket的数据发送

管道

​ redis是基于客户端-服务端模型和请求响应协议的TCP服务，客户端向服务器发送查询请求，并监听Socket返回，通常以阻塞模式等待服务器响应，服务端处理命令，并将结果返回。

​ redis管道技术可以在服务器未响应时继续向服务端发送请求，并最终一次性读取所有的服务器响应。

​ 管道技术可以大幅提高redis服务的性能。

分区

​ 分区是分割数据到多个redis实例的处理过程，每个实例只保存key的一个子集。

​ 优势：

  • 利用多台计算机的内存和值可以构造更大的数据库
  • 通过多核处理器和多台计算机可以扩展计算能力
  • 通过多台计算机和网络适配器可以扩展网络带宽

​ 劣势：

  • 涉及多个key操作通常不被支持，两个set映射到不同的redis实例时，不能对这两个set执行交集操作
  • 涉及多个key时redis事务无法使用
  • 使用分区时数据更复杂，会需要处理多个rdb/aof文件，并从多个实例和主机备份持久化文件
  • 增加和删除容量也会变得复杂
  • redis集群大多数支持在运行时增加和删除节点的透明数据平衡，但客户端分区和代理等其他系统则不支持，presharding可以解决一部分问题

分区类型有两种

范围分区

​ 映射一定范围内的对象到特定的redis实例，不过需要一个区间范围到实例的映射表，且这个表需要管理，而且还需要各种对象的映射表。

哈希分区

​ 用一个hash函数将key转换为数字，对数字取模，将其转化为0-3中的任意一个数字，就可以将这个整数映射到4个redis实例中的一个。
